# ENG-012 禁止会话内 follow-up 切换模型

- **status**: completed
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-021
- **created**: 2026-03-13

## 描述

当前 issue 在建立 session 后，follow-up 仍允许显式传入不同 `model`，后端会尝试在原会话上切模型。对于 ACP 多 agent 场景，这会让“模型切换”和“会话延续”语义混在一起，增加 session 复用错误和上下文不一致的风险。需要禁止用户在已建立会话后通过 follow-up 切换模型，并在前端直接锁定模型选择。

## 调查结果

- `followUpIssue()` 当前会用显式 `model` 覆盖 session 里的模型，并写回 issue session。
- ACP follow-up 会复用原 `externalSessionId`，再对同一 session 调用 `setSessionModel()`。
- `ChatInput` 当前在 issue 详情页仍允许用户修改 `selectedModel` 并随 follow-up 一起提交。
- `restart` 当前并不接收新的 `model` 参数，因此这次变更可以先聚焦在“禁止 follow-up 改模型”。

## 验证

- [x] follow-up 在已有 session 且模型变更时返回 `409`
- [x] ChatInput 在已有 session 时锁定模型选择
- [x] 定向 tests / lint 通过

## 实现结果

- `POST /issues/:id/follow-up` 现在会在已有 `externalSessionId` 且显式 `model` 与当前 session model 不一致时直接返回 `409`
- `ChatInput` 现在会在已有 session 时锁定模型选择，并且 follow-up 不再主动提交 `model`
- 新增中英文提示文案，明确“切换模型需要 restart”
