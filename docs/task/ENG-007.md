# ENG-007 ACP tool result 优先显示格式化输出

- **status**: completed
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-016
- **created**: 2026-03-13

## 描述

当前 ACP tool result 已经能成对落库并进入前端，但当 `rawOutput` 是结构化对象时，默认会整体 JSON 序列化显示，产品体验不佳。像 unified exec 返回的 `formatted_output`、`stdout`、`aggregated_output` 这类更适合人读的字段，应优先作为结果正文展示。

## 调查结果

- ACP result 正文目前由 `buildAcpToolResultContent()` 生成，优先级是 `content -> rawOutput(JSON) -> status text`。
- 这会导致 unified exec / shell 类工具结果显示整段 JSON，而不是直接显示命令输出文本。
- 前端工具卡片已经优先展示 `item.result.content`，因此只要后端把结果正文生成正确，前端无需额外特判。
