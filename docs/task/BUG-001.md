# BUG-001 未指定 root 目录时以自身所在目录为 root

- **status**: completed
- **priority**: P1
- **owner**: (未分配)
- **createdAt**: 2026-03-08 19:30

## 描述

当用户未配置项目目录时，`executeIssue` 和 `resolveWorkingDir` 使用 `process.cwd()` 作为工作目录的 fallback。`process.cwd()` 取决于启动进程时的当前目录，不可预测。应该使用 `ROOT_DIR`（应用自身所在目录）作为默认值。

**影响位置：**
- `apps/api/src/engines/issue/orchestration/execute.ts:77` — `process.cwd()` fallback
- `apps/api/src/engines/issue/utils/helpers.ts:41` — `resolveWorkingDir` 中的 `process.cwd()` fallback

## 进行时描述

修复未配置目录时的工作目录 fallback
