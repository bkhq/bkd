# BUG-011 ACP 工具结果被重复渲染为独立 tool 项

- **status**: completed
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-022
- **created**: 2026-03-13

## 描述

当前 ACP 某些工具调用会为同一个 `toolCallId` 先发一个占位 action，再发一个具体 action，最终前端表现为重复工具项。典型表现是 `Read File` 或 `Terminal` 先显示一条泛化工具，再显示一条带路径/命令的真实工具，破坏阅读顺序。

## 调查结果

- 问题根因不在前端 timeline，而在 ACP normalizer。
- 对 issue `1mwks59e` 的数据库日志检查显示，同一个 `toolCallId` 同时存在两条 action：
  - 一条泛化 `kind: tool` / `Terminal` / `Read File`
  - 一条具体 `kind: file-read` 或 `kind: command-run`
- 这是因为 ACP normalizer 在 `tool_call` 首次到来时立即发射 action，后续同一个 `toolCallId` 补充了更具体的信息后又再次发射 action。
- 受影响位置在 [normalizer.ts](/app/bkd/worktrees/n6ggd95o/l2801201/apps/api/src/engines/executors/acp/normalizer.ts)。

## 验证

- [x] 同一个 `toolCallId` 不再重复发射占位 action 与具体 action
- [x] `Read File` / `Terminal` 等工具在拿到具体信息后只显示一次 action
- [x] 定向 ACP tests / lint 通过

## 实现结果

- ACP normalizer 现在只会在拿到足够具体的信息后发射 action，并为每个 `toolCallId` 只发一次 action
- 新增回归测试，覆盖“占位 `Read File` -> 具体 `file-read`”不再重复的场景
- 历史 issue 中已经持久化的重复日志不会自动回写，但新的 ACP 运行不再产生该问题
