# ENG-003 Remove ACP engine

- **status**: completed
- **priority**: P2
- **owner**: claude
- **createdAt**: 2026-04-28

## Description

Remove the ACP engine entirely from the codebase. ACP is a virtual engine
type using protocol `acp:<agent>:<model>` that fronted Gemini, Codex, and
Claude through a shared Agent Communication Protocol layer. With the
dedicated `claude-code`, `claude-code-sdk`, and `codex` executors covering
the active engine surface, the ACP layer is no longer needed.

The change deletes the ACP executor module, all type-union and dispatch
references, the frontend ACP-specific timeline, related tests, and
documentation mentions of ACP / Gemini CLI support.

## ActiveForm

Removing the ACP engine and all virtual `acp:*` routing from BKD

## Out of scope

  Existing issues with `engineType` starting with `acp` will fail probe
  / spawn after this change; users must reassign them. (No data is
  destroyed; issues retain their stored engineType string.)
- Renaming, refactoring, or restructuring of remaining executors.
- The unrelated `IS_SANDBOX` / safe-env changes already staged in the
  working tree are preserved as-is (separate change).
