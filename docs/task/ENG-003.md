# ENG-003 ACP engine 支持基于 model 的多 agent 路由

- **status**: completed
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-010
- **created**: 2026-03-13

## 描述

将现有独立 `acp` engine 扩展为支持多个底层 ACP agent，并通过 `model` 字符串而不是新增数据库字段来区分具体 agent。首批接入 `@google/gemini-cli` 与 `@zed-industries/codex-acp`。

## 目标

- [x] 定义 `acp:<agent>:<model>` 的统一 model 标识格式
- [x] 在不改数据库 schema 的前提下完成 agent 选择与模型路由
- [x] 为 Gemini 与 Codex 提供各自的 command / availability / auth / model discovery 逻辑
- [x] 保持 issue/session/default model 流程兼容现有 `acp` engine

## 当前进展

- 已确认现有数据库中的 `model` 与默认模型设置都是普通字符串字段，可直接承载复合 model 标识。
- 已完成 `acp:<agent>:<model>` 解析与多 agent 路由，首批支持 Gemini 与 Codex。
