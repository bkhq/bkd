# ENG-005 ACP tool call 消息按文档配对到产品状态

- **status**: completed
- **priority**: P0
- **owner**: codex
- **plan**: PLAN-014
- **created**: 2026-03-13

## 描述

当前 ACP executor 只把 `tool_call` 与 `tool_call_update` 粗略映射为 `tool-use`，缺少基于 ACP 文档语义的 action/result 配对、输入输出保留与结果状态建模，导致前端虽然能看到工具消息，但还不到现有产品对 Claude/Codex 的展示水平。

## 调查结果

- ACP SDK 官方类型定义中，`tool_call` 提供 `title`、`kind`、`rawInput`、`rawOutput`、`content`、`locations`、`status`；`tool_call_update` 以增量形式更新同一组字段，并以 `toolCallId` 关联。
- 当前 `apps/api/src/engines/executors/acp/acp-client.ts` 仅把 `tool_call` 映射成普通 `tool-use`，`tool_call_update` 映射成 `Tool update: <status>`，没有 `isResult`、`toolAction`、`toolDetail.raw` 等完整结构。
- 现有产品侧的成对逻辑依赖 `toolCallId` + `isResult`，见 `apps/api/src/engines/issue/store/message-rebuilder.ts` 与前端 `use-chat-messages.ts`。
- Claude executor 已经按这种语义落地：工具 action 独立一条，结果独立一条，并通过 `toolCallId` 配对。
