# BUG-005 File browser rejects valid worktree root paths

- **status**: completed
- **priority**: P1
- **owner**: claude

## Problem

The file browser API at `/api/projects/:projectId/files/show?root=...` validated `root` against the project directory and BKD-managed worktree paths. Git worktrees created outside BKD's management (e.g., project repo worktrees) were rejected with 403 "Root is not a valid project or worktree directory".

## Root Cause

`isValidWorktreeRoot()` only checked `WORKTREE_BASE/<projectId>/` prefix. External git worktrees (e.g., `/app/bkd/worktrees/xfk81aye/jivzqdqg`) don't match this pattern and aren't subdirectories of the project directory either.

## Fix

Decoupled the file browser API from project context entirely:

1. Changed route from `/api/projects/:projectId/files/show` to `/api/files/show?root=<path>`
2. Removed project/worktree validation — `root` is now a required query param, only path traversal prevention remains
3. Updated frontend to pass `root` (from `rootPath ?? project.directory`) directly
4. Fixed `file-browser-store.toggle()` to clear `rootPath` when switching projects

## Files Changed

- `apps/api/src/routes/files.ts` — standalone root-based file browsing
- `apps/api/src/routes/api.ts` — mount at `/api/files` instead of project-scoped
- `apps/frontend/src/lib/kanban-api.ts` — new API signatures
- `apps/frontend/src/hooks/use-kanban.ts` — hook takes `root` instead of `projectId`
- `apps/frontend/src/components/files/FileBrowserDrawer.tsx` — resolve effective root
- `apps/frontend/src/pages/FileBrowserPage.tsx` — use project.directory as root
- `apps/frontend/src/stores/file-browser-store.ts` — toggle clears rootPath on project change
