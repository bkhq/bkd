# BUG-010 ACP assistant streaming 未落地导致前端无返回

- **status**: completed
- **priority**: P0
- **owner**: codex
- **plan**: PLAN-012
- **created**: 2026-03-13

## 描述

`acp` engine 在使用 `acp:codex:*` 等模型执行 follow-up 时，后端 issue debug log 能看到 agent 输出，但前端聊天面板没有任何 assistant 返回。现象表现为 `[BKD] Follow-up spawned ...` 之后 UI 不继续更新。

## 调查结果

- ACP client 当前把 `agent_message_chunk` 归一化为 `assistant-message`，但统一标记为 `metadata.streaming = true`。
- issue pipeline 的持久化阶段、ring buffer 阶段以及 SSE `log` 广播都直接跳过 `streaming` 条目，因此这些 assistant chunk 不会进入 DB，也不会通过普通 `log` 事件发给前端。
- ACP turn 结束时只会额外发出一个 `acp-prompt-result` system-message；该消息被标记为内部 `type: system`，在可见性过滤中默认隐藏，因此也不会作为用户可见回复出现。
- `data/logs/issues/dedeh172/debug.log` 已确认 follow-up 收到了 agent 文本输出与 `stopReason: "end_turn"`，说明问题不在子进程执行，而在 “ACP streaming -> 可见 assistant message” 的收尾桥接缺失。
