# WB-003 Refactor whiteboard AI: hidden sessions + MCP tools

- **status**: completed
- **priority**: P1
- **owner**: claude
- **createdAt**: 2026-04-16

## Description

The whiteboard currently creates a visible "[Whiteboard]" issue per project
that pollutes the regular issues list, and forces the AI into a brittle
`## heading` markdown contract. Refactor:

1. **Hide the bound issue** — add `issues.isHidden` column, filter from
   default listings, set 1 for whiteboard sessions.
2. **Whiteboard MCP tools** — expose `whiteboard-get-tree`, `whiteboard-add-node`,
   `whiteboard-update-node`, `whiteboard-delete-node`, `whiteboard-move-node`.
3. **Whiteboard system prompt** — instruct the AI to use MCP tools instead of
   emitting `## headings`; keep markdown parser as degraded fallback.
4. **Simplify frontend** — remove 4-way action enum, collapse to a single
   free-form ask, drop the post-turn parse-and-insert orchestration in favor
   of SSE-driven refetch.
