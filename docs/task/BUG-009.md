# BUG-009 模型发现链路缺少可诊断日志

- **status**: in_progress
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-011
- **created**: 2026-03-13

## 描述

当前 `/api/engines/available` 与相关模型发现链路在模型未返回时缺少足够的调试日志。尤其是 ACP 多 agent 场景下，单个 agent 的模型发现失败会被吞掉，最终只表现为模型列表为空或缺项，难以定位具体失败点。

## 调查结果

- `startup-probe.ts` 对 `executor.getAvailability()` 和 `executor.getModels()` 使用 `Promise.all()` + `Promise.allSettled()` 聚合，只在 engine 粒度记录总结果，不记录模型发现失败原因。
- `AcpExecutor.getModels()` 对各 agent 使用 `Promise.allSettled()`，失败分支直接返回空数组，没有日志。
- `queryScopedAcpModels()` / `queryAcpModels()` 成功时不记录 agent、模型数、默认模型；失败时也没有调用方级别上下文日志。
