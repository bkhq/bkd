# ENG-010 在 ACP engine 中接入 Claude agent

- **status**: completed
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-019
- **created**: 2026-03-13

## 描述

当前 `acp` engine 已支持 `gemini` 和 `codex` 两类 agent，模型格式为 `acp:<agent>:<model>`。本任务目标是为同一 `acp` engine 增加 `claude` agent 接入，继续沿用现有模型路由设计，不新增 engineType。

## 调查结果

- 当前 ACP agent registry 位于 `apps/api/src/engines/executors/acp/agents/`，接入新 agent 的主改动点是 `base.ts` 的 `AcpAgentId`、`index.ts` 的 registry，以及新增独立 `claude.ts` 定义文件。
- 现有 `gemini` 与 `codex` 都通过 `commandName` + `npxFallback` + `verify()` + `authStatus()` 接入，Claude 也适合复用这套结构。
- `@zed-industries/claude-code-acp` 当前 npm 版本为 `0.16.2`，可通过 `npx -y @zed-industries/claude-code-acp` 启动，命令名为 `claude-code-acp`。

## 实现结果

- 新增 `apps/api/src/engines/executors/acp/agents/claude.ts`
- 扩展 `AcpAgentId` 与 ACP agent registry，Claude 现在与 Gemini/Codex 并列接入同一个 `acp` engine
- 认证状态优先检查 `ANTHROPIC_API_KEY`，其次检查本地 `~/.claude/.credentials.json`
- 为 probe 结果增加版本键，确保旧 discovery 缓存在新增 agent 后自动失效并重新 live probe

## 验证
