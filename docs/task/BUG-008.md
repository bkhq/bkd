# BUG-008 ACP Codex model id 含 `/` 导致 400

- **status**: completed
- **priority**: P1
- **owner**: codex
- **created**: 2026-03-13

## 描述

ACP 多 agent 路由中，Codex 返回的模型 ID 形如 `gpt-5.4/high`，组合为 `acp:codex:gpt-5.4/high` 后会被 API 的 `model` 正则校验拒绝，导致执行和 follow-up 请求返回 400。

## 调查结果

- `executeIssueSchema` 和 `createIssueSchema` 当前只允许 `[\w.:-]`，不允许 `/`。
- `follow-up` 的 multipart 分支仍使用旧正则，既不允许 `/`，也不允许 `:`。

- [x] `bunx eslint apps/api/src/routes/issues/_shared.ts apps/api/src/routes/issues/message.ts`
- [x] 正则验证：`acp:codex:gpt-5.4/high` 可通过 execute/create/follow-up 的 model 校验
