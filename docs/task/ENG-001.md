# ENG-001 Claude executor 添加 --debug-file 参数

- **status**: completed
- **priority**: P1
- **owner**: claude-opus
- **created**: 2026-03-08

## 描述

Claude Code CLI 支持 `--debug-file <path>` 参数，将调试日志写入指定文件。当前 `ClaudeCodeExecutor.createBaseBuilder()` 未添加此参数，导致 BKD 派生的 Claude 进程无法查看调试日志，排查问题困难。

## 调查结果

- 修改范围：仅 `apps/api/src/engines/executors/claude/executor.ts`
- `createBaseBuilder()` 方法构建 CLI 参数，需在此添加 `--debug-file`
- `ExecutionEnv` 提供 `issueId`，可用于命名日志文件
- 日志目录已有约定：`data/logs/`（参见 `logger.ts`）
- 调试日志文件路径建议：`data/logs/claude-debug-{issueId}.log`

## 验收标准

- [ ] Claude 进程启动时携带 `--debug-file` 参数
- [ ] 调试日志写入 `data/logs/` 目录
- [ ] 文件名包含 issueId 以便区分不同 issue
